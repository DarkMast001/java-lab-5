Objects.requireNonNull(obj) - для стандартной проверки в релизном коде, всегда бросает исключение NullPointerException, соответсвенно можно проверить через try catch
assert obj != null - для отладочной проверки, а не для защиты данных (нацелен на ошибку программиста), ЕСЛИ ВКЛЮЧЕН, то бросает исключения AssertionError, но надо включать с помощью -ea, то есть не всегда работает
